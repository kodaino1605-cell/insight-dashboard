# Glossary（用語集）

> 最終更新日: 2026-06-16
> 関連ファイル: [functional-design.md](./functional-design.md) / [architecture.md](./architecture.md)

## 1. 本書の目的

世代区分・プロダクト固有用語の定義を一意化し、コード・ドキュメント間での解釈のブレを防ぐ。

---

## 2. 世代区分の定義

世代の生年範囲は複数の定義が存在するため、本ドキュメントで採用する基準を「本プロダクトにおける定義」として固定する。厳密な学術定義ではなく、実務上広く参照される目安を採用する。

| 世代名 | enum値 | 生年の目安 | 2026年時点の年齢目安 | 備考 |
|---|---|---|---|---|
| α世代 | `gen_alpha` | 2010年頃〜2024年頃 | 2-16歳 | 大部分が未成年。本プロダクトでは「観察対象」としての位置づけが中心 |
| Z世代 | `gen_z` | 1995年頃〜2009年頃 | 17-31歳 | JKトレンドの中心的な対象層と重なる |
| ミレニアル世代（Y世代） | `gen_millennial` | 1980年頃〜1994年頃 | 32-46歳 | |
| X世代 | `gen_x` | 1965年頃〜1979年頃 | 47-61歳 | 「バブル・団塊Jr世代」と年代が一部重複（下記参照） |
| バブル・団塊Jr世代 | `gen_bubble_dankai_jr` | 1960年頃〜1971年頃 | 55-66歳 | 日本独自の世代区分。「バブル世代（〜1965年頃生まれ）」と「団塊Jr世代（〜1971年頃生まれ）」を本プロダクトでは便宜上一括グルーピングする |
| シニア世代 | `gen_senior` | 1959年頃以前 | 67歳以上 | |

### 年代重複の扱い

「バブル・団塊Jr世代」（1960年頃〜1971年頃）と「X世代」（1965年頃〜1979年頃）は国際的な世代区分と日本独自の世代区分が混在しており年代が重複する。1記事が複数の `generation_tags` を同時に持てる設計（[functional-design.md](./functional-design.md) §3）のため実用上の支障は小さいが、世代分類プロンプトに渡す基準として上表の生年範囲を確定値として使用する。

## 3. プロダクト固有用語の定義

| 用語 | 定義 |
|---|---|
| ビッグニュース | 特定世代に限らず社会全体にとって重要度が高いと判定されたニュース。カテゴリの一つであり世代タグとは独立した分類軸 |
| JKトレンド | 女子高生を起点とした流行・関心事を分析するカテゴリ。MVPでは一般ニュースソースから推測的に構成するプレースホルダーであり、SNS発の一次情報を直接捕捉しているわけではない（[functional-design.md](./functional-design.md) §6） |
| 背景感情 | そのニュースに対して人々が反応している心理的・感情的な理由（驚き、不安、共感、反発等） |
| 欲望 | ニュースの背後にある、人々の満たされていないニーズや願望 |
| 時代の流れ | そのニュースが示す、より大きな社会的・構造的変化の方向性 |
| マーケティング示唆 | 商品企画・マーケティング活動に転用可能な実践的な気づき・提案 |
| バッチ生成 | 毎朝8:00に実行される、RSS収集からAI分析・保存までの一連の自動処理（リアルタイム分析と対比される概念） |
| カテゴリ | トップページのカード単位（ビッグニュース/6世代/ビジネスマン/JKトレンドの計9種）。世代タグ（記事の属性）とは別概念であり、表示上の分類軸である |

---

## 4. 用語の使用ルール

コード内の識別子（enum/Union型のリテラル値）は本グロッサリーの日本語名称と1対1で対応させる。新しい用語・カテゴリを追加する場合は、必ず本ファイルを先に更新してから実装する（[CLAUDE.md](../CLAUDE.md) のスペック駆動運用ルールに従う）。
